# Proyecto Web Modular: Landing Page, Registro, Login y Home

##  Descripción General

Este proyecto consiste en el **diseño y desarrollo de una aplicación web modular** compuesta por **cuatro vistas principales**:

1. **Landing Page**  
2. **Registro (Sign Up)**  
3. **Login (Inicio de sesión)**  
4. **Home (Área principal de usuario)**  

El objetivo principal es **integrar los conocimientos de Desarrollo Web en Entorno Cliente (DWEC)** y **Diseño de Interfaces Web (DIW)**, aplicando **buenas prácticas profesionales** tanto a nivel técnico como visual.

---

## Objetivos

- Aplicar el **modelo de arquitectura MVC (Modelo–Vista–Controlador)** para estructurar el código.  
- Mantener una **guía de estilo coherente y profesional**, asegurando consistencia visual.  
- Simular la gestión de datos mediante **arrays u objetos en JavaScript**, almacenados temporalmente en memoria.  
- No se requiere ningún **backend** ni **base de datos real**.  
- Garantizar un producto **estructurado, legible, funcional y estéticamente cuidado**.

---

## Tecnologías y Herramientas

- **HTML5** → Estructura semántica del contenido.  
- **CSS3 / SASS** → Diseño, maquetación y guía de estilos coherente.  
- **JavaScript** → Lógica de interacción, controladores y simulación de datos.  
- **Modelo MVC** → Separación de responsabilidades:  
  - **Modelo:** Gestión de datos simulados.  
  - **Vista:** Interfaz y renderización dinámica.  
  - **Controlador:** Comunicación entre modelo y vista.  
- **Git / GitHub** → Control de versiones y colaboración.

## Guía de Estilo y Diseño

- Uso de una **paleta cromática coherente** y tipografías legibles.  
- Diseño **responsive** adaptable a distintos dispositivos.  
- Enfoque en la **usabilidad** y la **experiencia de usuario (UX)**.  
- Cumplimiento de **buenas prácticas de accesibilidad (WAI-ARIA)**.  

---

## Funcionamiento General

- Los **usuarios pueden registrarse** mediante un formulario que almacena sus datos en memoria (simulación con arrays/objetos).  
- El **inicio de sesión** valida las credenciales con los datos almacenados.  
- Una vez logueado, el usuario accede a la **vista Home**, donde se muestra información personalizada.  
- En caso de error (credenciales incorrectas, campos vacíos, etc.), el sistema muestra **mensajes de validación** claros y accesibles.
